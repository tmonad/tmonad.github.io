Get Started

What is tmonad

tmonad stands for traffic monad, it is an easy-to-customize HTTP Gateway

tmoand is like Apache traffic server or Varnish, it comes with cache mechanism and routing remap capabilities. Also you can easily custommize it with Haskell programming languages. It follows the design of xmonad, providing hooks for plugins to further customize it.

Why should I use tmonad

tmonad is reasonably fast, but easy to add plugin and fine tune

Most existing reverse proxy application resorts to event-driven programming to pursue for the fastest speed. However, event-driven programming style would become difficult to comprehend when the repository grows large. We believe the powerful GHC runtime with event-driven based IO Manager and threading programming abstraction could resolve this issue, and on the same par with the existing solution on speed.
